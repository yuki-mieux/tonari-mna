## 🎯 プロダクトビジョン

### ミッション

**「M&A仲介者の隣にいる、最高の相棒」**

**ゴール: ミーティング終了直後に、ノンネームとIMが出来上がっている世界**

### TONARIプロダクトファミリーにおける位置づけ

```
                    TONARI（コア技術）
                   リアルタイム音声 → 構造化抽出
                          │
          ┌───────────────┼───────────────┐
          ↓               ↓               ↓
    面接支援版        ハラスメント版      M&A版 ← このプロジェクト
   (人材紹介業界)     (大手企業管理職)   (M&A仲介)
```

### 共通の技術本質

**「リアルタイム音声から、業務に必要な構造化情報を抽出し、意思決定を支援する」**

| プロダクト | 抽出する情報 | 提供する価値 |
|-----------|-------------|-------------|
| TONARI（本家） | 求職者の適性・マッチング要素 | 見落とし防止・判断の質向上 |
| ハラスメント版 | パワハラリスク発言 | 無自覚リスクの可視化・言い換え提案 |
| **M&A版** | **案件情報（IM/ノンネーム項目）** | **資料作成工数の激減・属人化解消** |

## 📊 M&A項目抽出フォーマット（約35項目）

### 抽出項目の構成

```
IM抽出項目:
├── 会社概要
│   ├── 会社名、所在地、設立年、資本金
│   ├── 事業内容、従業員数、組織体制
│   └── 沿革、代表者プロフィール
├── 事業内容
│   ├── 主要サービス/製品
│   ├── 取引先・顧客構成
│   ├── 競合優位性、強み・弱み
│   └── 業界動向、市場ポジション
├── 財務ハイライト
│   ├── 売上高推移（3〜5期）
│   ├── 営業利益、経常利益
│   ├── 純資産、調整後純資産
│   └── 主要KPI
├── 譲渡条件
│   ├── 譲渡スキーム（株式譲渡/事業譲渡）
│   ├── 希望価格
│   └── 譲渡理由、希望条件
└── その他
    ├── 従業員の処遇
    ├── 役員の残留意向
    └── デューデリ留意事項
```

### IM仕様（水野氏ヒアリングより）

| 項目 | 内容 |
|------|------|
| **ページ数** | 約30〜35ページ |
| **デザイン** | 会社ごとの独自性不要、CXOグループ標準テンプレート1枚でOK |
| **レイアウト** | 決め打ちでOK |
| **可変部分（手動修正必要）** | 6, 7, 8, 11, 12, 13ページ付近（調整後純資産、不良性資産等） |
| **固定部分** | 財務ハイライト、譲渡スキーム（株式譲渡/事業譲渡の2パターン） |

## 🔄 開発方針

### DDファイル管理システムとの関係

**方針転換決定（12/24）:**
- DDファイル管理システムの開発は一旦動作確認レベルで留める
- M&Aの最上流工程（ノンネーム・IM生成）に注力

**理由:**
- DDファイル管理は案件が土俵に乗ってから使うもの（下流工程）
- ノンネーム・IMは案件数が圧倒的に多い（上流工程）
- 上流を押さえることで粘着性が高まり、下流への移行もスムーズになる

### 人材紹介版からの転用

土戸氏が開発済みの「人材紹介業界向け求人票構造化ツール」をM&A版に転用：

```
人材紹介版:
  求人票（バラバラ形式）→ AI抽出 → ポータルス管理項目へマッピング

M&A版:
  ヒアリング/決算書/Web（バラバラ形式）→ AI抽出 → IM項目へマッピング
```
