# TONARI for M&A - 画面設計・UI仕様書

## 1. 設計原則

### 1.1 コンセプト
**「話すことに集中できる」UI**

- 面談中は会話に集中、システムが裏で情報を収集
- 必要な情報が必要なタイミングで目に入る
- 操作は最小限、視線移動も最小限

### 1.2 デザイン原則

| 原則 | 説明 |
|------|------|
| 情報の階層化 | 重要度に応じた視覚的優先順位 |
| 邪魔しない | 会話を遮らないサジェスト表示 |
| 一目でわかる | 進捗・状態が瞬時に把握可能 |
| 操作レス | 可能な限り自動化、手動操作は最小限 |

## 3. コンポーネント詳細

### 3.1 ヘッダー

```
┌─────────────────────────────────────────────────────────────┐
│  🏠 TONARI    [株式会社○○ ▼]    [● 録音中] 32:15:03   ⚙️   │
└─────────────────────────────────────────────────────────────┘
```

| 要素 | 説明 |
|------|------|
| ロゴ | クリックでダッシュボードへ |
| 案件セレクター | 現在の案件名、ドロップダウンで切替 |
| 録音ステータス | 録音中は赤丸点滅、停止中はグレー |
| 経過時間 | HH:MM:SS形式 |
| 設定アイコン | 音声設定、表示設定等 |

#### 会話カードの構成

| 要素 | 説明 |
|------|------|
| タイムスタンプ | 発言時刻（MM:SS） |
| 話者ラベル | 「あなた」「相手」（将来的に名前設定可） |
| 発言テキスト | リアルタイム更新 |
| 重要マーク | ★ = システムが重要と判断した発言 |
| ピン留めボタン | 手動でマーク可能 |

#### インタラクション

- **スクロール**: 自動追従（新しい発言を常に表示）
- **ピン留め**: ホバーで表示、クリックでトグル
- **検索**: 🔍アイコンで会話内検索

### 3.3 右パネル：情報タブ

#### タブ構成

| タブ | 内容 | 優先度 |
|------|------|--------|
| 抽出 | 抽出された情報の一覧 | 常時表示 |
| 分析 | 構造分析・買い手像 | 情報蓄積後 |
| 提案 | 水野メソッドサジェスト | 常時表示 |

#### 抽出項目の状態

| 状態 | アイコン | 意味 |
|------|----------|------|
| 取得済 | ✓（緑） | 確信度高く抽出完了 |
| 推定中 | △（黄） | 抽出したが確信度低 |
| 未取得 | ○（グレー） | まだ情報なし |

- **項目クリック**: 根拠となる会話箇所にジャンプ
- **セクション折りたたみ**: ▼クリックで開閉
- **手動編集**: 項目をクリックして値を修正可能

## 5. 通知・フィードバック

### 5.1 リアルタイム通知

| 通知タイプ | 表示方法 | トリガー |
|------------|----------|----------|
| 情報抽出 | 右パネルの項目がハイライト | 新情報抽出時 |
| 重要発言 | 会話カードに★マーク | 重要ワード検知時 |
| サジェスト更新 | 提案タブにバッジ | 新しい提案生成時 |
| リフレーミング | 提案タブに強調表示 | ネガティブワード検知時 |

### 5.2 トースト通知

```
┌─────────────────────────────┐
│ ✓ 売上情報を抽出しました   │
│   「年商3億円」            │
└─────────────────────────────┘
```

- 右下に3秒間表示
- 複数の場合はスタック
- クリックで詳細へジャンプ

## 7. アクセシビリティ

### 7.1 キーボード操作

| キー | 動作 |
|------|------|
| Space | 録音開始/停止 |
| Tab | フォーカス移動 |
| 1/2/3 | タブ切替（抽出/分析/提案） |

| Esc | ダイアログ閉じる |

### 7.2 カラーコントラスト

- WCAG AA準拠
- 色だけに依存しない情報伝達（アイコン併用）
